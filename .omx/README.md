# Repo-Local Workflow Pack

This is a low-risk, repo-local workflow layer inspired by `oh-my-codex`.

It does not try to replace the operator's global Codex installation. Instead, it gives this repository a stable working shape:

- clarify
- plan
- execute
- verify
- architecture review
- architecture design
- architecture retro

Use the command guides in `.omx/commands/` as project-local operating conventions.

Persistent documents belong in `docs/plans/`.
Ephemeral runtime notes belong in `.omx/state/` or `.omx/scratch/` if you create them locally.

Architecture prompt assets live in:

- `docs/prompts/`
- `docs/reviews/`
- `docs/adr/`
- `.omx/templates/`
