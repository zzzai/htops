# Hetang Manager Customer Profile Design

**Goal**

把现有“资料卡式”顾客画像收敛成店长可直接执行的经营画像，重点回答“这个顾客值不值得经营、现在处在什么阶段、该怎么跟进、今天先做什么”。

**Current Problem**

- 现有输出字段完整，但更像分析台账，不像店长班前会或客服跟进单。
- 同一份画像里混合了过多底层分析描述，重点不够前置。
- 无偏好或无附加消费明细时会出现“暂无可识别明细”式表达，影响管理层阅读观感。

**Design**

店长版顾客画像固定收敛成 6 段：

1. `一句话判断`
2. `当前状态`
3. `顾客价值`
4. `偏好与习惯`
5. `风险与机会`
6. `今日先抓`

**Rules**

- 不新增业务口径，只重组现有顾客画像能力。
- 不输出 `N/A`。
- 对于缺少可靠值的偏好块，整行隐藏，不用“暂无可识别明细”填充。
- `今日先抓` 最多 3 条，语气必须像店长可直接布置的动作。
- 茶饮 / 餐食 / 副项等专项问法保持现有独立输出，不强行改成店长版长文。

**Data Sources**

- `fact_member_current`
- `fact_member_cards_current`
- `fact_consume_bills`
- `fact_tech_up_clock`
- `fact_tech_market`
- `mv_customer_profile_90d` / 客群快照回退链

**Rendering Strategy**

- 保留现有顾客等级、生命周期、经营分层、沉默风险、复购概率、支付结构诊断、团购承接链路等判断逻辑。
- 将原 `店长动作建议` 8 行建议压缩成：
  - 风险与机会：解释“为什么现在值得跟/为什么有风险”
  - 今日先抓：输出 1~3 条动作
- 让 `一句话判断` 成为整份画像的第一结论，而不是把判断藏在中间。
