# Weekly HQ Package Rollout Design

日期：2026-04-20
状态：approved
范围：总部周报正文、周图、自动推送任务、店长使用教程

## 背景

当前周图已经基本满足“手机端可读 + 本周 vs 上周 + 5店对比”的要求，但整套总部周包还没有完全达到正式推送状态：

- `weekly report` 正文仍带有较重的旧指标叙事，如 `团购7天复到店`
- 周报正文与周图的核心指标口径还没有完全统一
- 调度层只有 `enabled + time`，没有“从哪一周开始正式启用”的安全闸门
- 今天是周一，如果直接打开自动发送，存在当天误发风险
- 还缺一份给店长看的使用教程，帮助门店理解“看什么、怎么行动”

目标不是继续堆内容，而是把 `周报正文 + 周图 + 调度配置 + 使用教程` 组合成一套可以正式交付总部群、同时能让店长落地执行的经营包。

## 设计目标

本次改造必须同时满足 5 点：

1. 今天不发送
2. 下周一开始可以自动正常发送
3. 周报正文与周图使用同一组核心经营信号
4. 总部看到的是“判断 + 动作”，店长看到的是“盯盘 + 执行”
5. 不引入新数据库，不新增第二条发送链路

## 方案选择

### 方案 A：只改文案，不改调度

优点：

- 改动最少

缺点：

- 仍然可能今天误发
- 上线动作依赖人工记忆，不安全

### 方案 B：只加调度闸门，不改正文

优点：

- 能解决误发风险

缺点：

- 正文和周图仍然像两套东西
- 总部收到后的经营价值仍然不够稳定

### 方案 C：统一口径 + 增加起始日期闸门 + 配套教程

优点：

- 一次把“能发”“该发什么”“收到后怎么用”三件事一起收口
- 不新增系统复杂度
- 最符合当前“配置好任务，不今天发送，下周正式上线”的目标

缺点：

- 需要同步改 `config / schedule / weekly-report / docs`

推荐采用方案 C。

## 核心设计

### 1. 周包核心指标统一

总部周包统一围绕以下信号展开：

- 营收
- 客流
- 加钟率
- 点钟率
- 新增会员
- 本周新增储值

其中：

- `营收` 继续作为主结果指标
- `客流 / 加钟率 / 点钟率 / 新增会员 / 本周新增储值` 作为判断链路
- `团购7天复到店` 不再作为总部周报主叙事

这与现有周图保持一致，让正文和图片读出来是一套语言。

### 2. 周报正文结构

周报正文继续保持手机端友好的短结构，但调整为更明确的总部经营表达：

- `一、整体判断`
  - 本周营收、客流、加钟率、点钟率、新增会员、本周新增储值的周环比
  - 直接指出增长来自哪几家店、回撤集中在哪几家店
- `二、关键动作`
  - 只保留 3 条总部级动作
  - 一条复制强店动作
  - 一条修复弱店断点
  - 一条周中复盘机制
- `三、门店动作`
  - 每店保持短结构，但内容必须围绕当前周图信号差异生成
  - 不再大面积复用旧的团购复到店话术
- `四、口径说明`
  - 保留必要说明，避免正文太长

### 3. 自动发送起始日期闸门

为周报、周图各增加独立起始日期配置：

- `reporting.weeklyReportStartDate`
- `reporting.weeklyChartStartDate`

规则：

- 未配置时，保持现有行为不变
- 配置后，仅当 `本地日期 >= startDate` 且满足原有周一时间条件时，任务才视为 due
- 这样可以提前把任务与发送开关都配好，但不会在今天误发

本次默认上线日期设为：`2026-04-27`

### 4. 任务启用策略

总部正式周包采用以下配置：

- `sendWeeklyReportEnabled = true`
- `sendWeeklyChartEnabled = true`
- `weeklyReportStartDate = 2026-04-27`
- `weeklyChartStartDate = 2026-04-27`

这样可以保证：

- 今天不会发
- 下周一起，文本周报和周图会按既定顺序自动执行

### 5. 店长使用教程

新增一份店长教程文档，聚焦 4 件事：

- 日报怎么看
- 周报怎么看
- 周图怎么看
- 召回命令怎么配合使用

教程不是介绍系统，而是给店长明确行动方法：

- 每天先看哪三个信号
- 周一拿到周包后怎么拆动作
- 哪种情况该查 `/hetang reactivation summary`
- 哪种情况该直接跟进高价值客户

## 数据与系统边界

本次不引入新数据库，也不新增新服务。

继续复用：

- `mart_daily_store_reports`
- `reporting-service`
- `weekly-report.ts`
- `weekly-chart-image.ts`
- `schedule.ts`
- `sync-orchestrator.ts`

新增的只是：

- 配置字段
- 调度 gating 逻辑
- 正文渲染逻辑调整
- 使用文档

## 验收标准

本次上线完成后，至少满足：

1. 当前 `2026-04-20` 不会因为配置变更而发送周报或周图
2. `2026-04-27` 起，周报与周图都能进入自动调度
3. 周报 dry-run 中不再把 `团购7天复到店` 作为总部主叙事
4. 周报正文和周图能围绕同一组经营信号表达
5. 店长教程能直接发给门店使用，而不是技术说明文档
