# Store Manager Bot Guide Poster Design

日期：2026-04-20
状态：approved
范围：店长企微问数使用教程图片

## 背景

纯文字教程对店长来说阅读成本太高。

这次目标不是继续写文档，而是做一张可以直接发给店长看的 `手机竖版教程图`。

用户已经明确要求：

- 只保留 3 个问题：
  - `可以问什么`
  - `怎么问`
  - `注意事项`
- 要做成 `一个图片`
- 内容要简单、易懂、适合店长

## 设计目标

这张图必须满足：

1. 手机上直接看得清
2. 第一眼就知道“能问什么”
3. 例句要短，能直接照着发
4. 不讲系统原理
5. 不要像说明书，要像店长培训海报

## 版式方向

采用 `手机竖版长图`。

原因：

- 最适合企微里直接转发
- 店长上下滑一遍就能看完
- 比横版更适合手机

## 信息结构

整张图只分 3 块：

### 1. 可以问什么

只放高频场景：

- 昨日日报
- 营收、客流、点钟率、加钟率
- 风险和建议
- 高价值客户跟进

### 2. 怎么问

只放短句示例，句子必须像店长真实会发的话：

- `迎宾店昨天日报`
- `迎宾店昨天营收`
- `迎宾店昨天有什么风险和建议`
- `迎宾店今天高价值客户跟进情况`

### 3. 注意事项

只保留最关键的 4 条：

- 说清门店
- 说清时间
- 说清指标
- 一次只问一件事

群聊场景单独补一句：

- 群里先点机器人，再发问题

## 视觉方向

风格要求：

- 干净
- 高级
- 易读
- 不花

视觉语言：

- 浅底
- 深字
- 蓝色做重点
- 卡片式分区
- 标题大，例句更大

不做：

- 复杂插画
- 过多颜色
- 密密麻麻的小字

## 技术实现

继续走现有成熟链路：

- `SVG 渲染`
- `Chrome headless 截图`
- 输出 `PNG`

不引入新依赖，不走外部图片服务。

## 验收标准

完成后至少满足：

1. 图片是一张手机竖版长图
2. 图上只保留 `可以问什么 / 怎么问 / 注意事项`
3. 每条示例都能直接给店长照着发
4. 手机上无需放大就能看清主内容
