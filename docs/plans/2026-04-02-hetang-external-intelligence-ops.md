# Hetang External Intelligence Ops

## 1. 目标

这份文档面向当前 `hetang-ops` 的 HQ 外部情报链路，目标不是做“新闻摘要”，而是为单一足疗品牌提供一份可用于当天经营判断的总部情报简报。

当前定位：

- 单品牌做深
- 总部先用
- 服务店长当天营收提升
- 只保留能转成经营判断的外部信号

## 2. 当前产物

系统当前每天生成一份：

- `何棠 HQ 外部情报简报`

固定输出要求：

- 标题
- 标签
- 时间
- 来源
- 段落摘要
- 经营提示

若当日没有足够高质量事件，允许少发，不允许拿垃圾内容补满。

## 3. 调度与命令

当前默认调度：

- `08:50` 生成 HQ 外部情报简报

总部命令面：

```bash
/hetang intel run
/hetang intel latest
/hetang intel issue ext-brief-2026-04-03
/hetang intel sources
```

说明：

- `run` 适合人工立即重建
- `latest` 适合 HQ 复核当天简报
- `issue` 适合复盘历史期次
- `sources` 适合检查分层配置是否偏了

## 4. 质量门槛

只有满足以下条件的内容才能进入最终简报：

1. 是事件，不是观点集合
2. 有可靠时间
3. 有完整中文摘要
4. 经过去重聚合，不重复写同一事件
5. 来源达到 `s / a`，或由 `b` lead 成功回填确认

当前明确拦截：

- 课程宣传
- 咨询软文
- 旧闻翻炒
- 无可靠时间
- 单纯热榜搬运

## 5. 分层规则

源分层：

- `s`: 官方、监管、平台公告、公司官方
- `a`: 主流媒体、垂直行业媒体
- `b`: 搜索结果、热榜、社媒搬运
- `blocked`: 软文、课程、会议宣传、SEO 聚合

落地规则：

- `s / a` 可以直接进入候选池
- `b` 只作为 lead，不直接进最终 Top 列表
- `blocked` 直接淘汰

## 6. 排障顺序

若 HQ 反馈“内容差”或“没收到”，按这个顺序看：

1. `external_source_documents`
2. `external_event_candidates`
3. `external_event_cards`
4. `external_brief_issues`
5. HQ 投递目标

判断原则：

- 如果源文档就差，先修源
- 如果候选全被挡掉，先修规则或时间判断
- 如果 cards 正常但 issue 空，查组装阈值
- 如果 issue 正常但没送到企微，查 channel 和 target

## 7. 对产品演进的意义

这条链路的作用不是替代店长日报，而是给 HQ 提供一个“外部变化到经营判断”的前置层。成熟后再把高置信度信号下翻成店长当天动作建议。

正确演进顺序：

1. 先把 HQ 简报质量做实
2. 再把 HQ 判断做实
3. 再把判断翻译成店长动作
4. 最后做动作效果回流
