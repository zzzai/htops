# Architecture Reviews

Use this directory for durable review and retro artifacts.

Recommended naming:

- `YYYY-MM-DD-<topic>-review.md`
- `YYYY-MM-DD-<topic>-retro.md`

Use `docs/plans/` when the output is a forward-looking design or implementation blueprint.
Use `docs/adr/` when the output is a durable architectural decision with a chosen option.
